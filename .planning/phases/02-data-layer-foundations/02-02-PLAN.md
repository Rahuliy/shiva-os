**Part A — Create `date-filter.js` at project root:**

An ES module that renders and manages the shared date filter bar. Exports one function: `initDateFilter(targetId, onChange)`.

Constants:
- `STORAGE_KEY = 'artha-date-filter'`
- `DEFAULT_PRESET = '1Y'` (per user decision — 1 year on first load)
- `PRESETS = ['1W', '1M', 'YTD', '3M', '1Y']` (standard presets as buttons)

`initDateFilter(targetId, onChange)`:
1. Get the container element by `targetId`. If not found, return early.
2. Read saved preset from `localStorage.getItem(STORAGE_KEY)`. Default to `'1Y'` if no saved value.
3. Render the filter bar HTML into the container:
   - A `div.date-filter-bar` with `role="tablist"` and `aria-label="Date range filter"`
   - One `button.date-filter-btn` per preset with `data-preset`, `role="tab"`, `aria-selected`
   - The saved/default preset button gets class `active`
   - A "Custom" button that toggles a compact inline date range picker
4. Custom range: when "Custom" is clicked, show/create two `<input type="date">` elements (start and end) inline next to the buttons. When both dates are set, fire `onChange('custom', { start: new Date(startInput.value + 'T00:00:00'), end: new Date(endInput.value + 'T00:00:00') })`. Append `'T00:00:00'` to the date-only string to avoid UTC-midnight parsing trap.
5. Wire click handlers on each preset button:
   - Update active state (toggle `active` class and `aria-selected`)
   - Store preset in `localStorage.setItem(STORAGE_KEY, preset)`
   - Call `onChange(preset)` with the selected preset string
   - Hide custom date inputs when a non-custom preset is selected
6. Fire `onChange(current)` immediately on init so the view loads with the saved range.

CSS for the filter bar — embed as a `<style>` element injected once by the module (check if already injected to avoid duplicates). Styles must use the project's design system tokens:
- `.date-filter-bar`: `display: flex; align-items: center; gap: 2px; padding: 6px 36px; background: var(--topbar-bg, rgba(14,8,5,0.85)); border-bottom: 1px solid var(--border-0, #1a1208); position: sticky; top: 50px; z-index: 190; backdrop-filter: blur(12px);`
- `.date-filter-btn`: `font-family: var(--mono, 'IBM Plex Mono', monospace); font-size: 11px; letter-spacing: 0.14em; color: var(--text-3, #4a3418); background: none; border: 1px solid transparent; border-radius: 1px; padding: 3px 10px; cursor: pointer; transition: color 0.15s, border-color 0.15s, background 0.15s; text-transform: uppercase;`
- `.date-filter-btn:hover`: `color: var(--text-2, #7a5a2c); border-color: var(--border-1, #2a1e12);`
- `.date-filter-btn.active`: `color: var(--gold, #c9973a); border-color: var(--gold-dim, #8a6a28); background: var(--gold-bg, rgba(201,151,58,0.08));`
- `.date-filter-custom-inputs`: `display: flex; align-items: center; gap: 6px; margin-left: 8px;` — contains the two date inputs styled to match dark mode (dark background, gold border on focus)
- Date input styling: dark background, monospace font, match project aesthetic

Also add skeleton show/hide helper functions:
```javascript
function showSkeleton(containerId) {
  const sk = document.getElementById(containerId + '-skeleton');
  const ct = document.getElementById(containerId + '-content');
  if (sk) sk.style.display = 'block';
  if (ct) ct.style.display = 'none';
}
function hideSkeleton(containerId) {
  const sk = document.getElementById(containerId + '-skeleton');
  const ct = document.getElementById(containerId + '-content');
  if (sk) sk.style.display = 'none';
  if (ct) ct.style.display = 'block';
}
```
