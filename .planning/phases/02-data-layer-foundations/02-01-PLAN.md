Create `artha-utils.js` at the project root as an ES module (`export function ...`).

**Currency utilities (INFRA-09):**

1. `toCents(dollars)` — Converts float dollar amount to integer cents using `Math.round(dollars * 100)`. Returns an integer. Example: `toCents(10.30)` returns `1030`.

2. `fromCents(cents)` — Converts integer cents back to float dollars via `cents / 100`. Use only when interfacing with external APIs expecting floats.

3. `formatCurrency(cents)` — Formats integer cents as a whole-number US dollar string using a module-level `Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD', minimumFractionDigits: 0, maximumFractionDigits: 0 })` constant (construct the formatter ONCE as `USD_FORMATTER`, reuse on every call — `Intl.NumberFormat` construction is expensive). Example: `formatCurrency(123456)` returns `"$1,235"`. Negative values display with a minus sign: `formatCurrency(-45000)` returns `"-$450"` — per user decision, Mint/Robinhood style with whole numbers only.

**Date utilities (supports INFRA-10 date filtering):**

4. `normalizeToLocalDate(input)` — Accepts a Date object or ISO string. Extracts the LOCAL calendar day using `getFullYear()`, `getMonth()`, `getDate()` (NOT UTC methods). Returns `new Date(y, m, d)` — midnight in local timezone. This prevents the UTC cross-midnight bug where Eastern US users see Feb 22 transactions bucketed under Feb 21.

5. `isSameLocalDay(a, b)` — Returns true if two dates/strings fall on the same local calendar day by comparing `normalizeToLocalDate(a).getTime() === normalizeToLocalDate(b).getTime()`.

6. `localDateString(d)` — Returns a `YYYY-MM-DD` string using LOCAL date components (not UTC). Safe for display labels and localStorage keys.

7. `getDateRange(preset, customRange)` — Accepts a preset string (`'1W'`, `'1M'`, `'YTD'`, `'3M'`, `'1Y'`, `'custom'`) and optional `{start, end}` for custom. Returns `{start: Date, end: Date}` normalized to local midnight. Implementation:
   - `'1W'`: today minus 7 days
   - `'1M'`: today minus 1 month
   - `'3M'`: today minus 3 months
   - `'YTD'`: January 1 of current year
   - `'1Y'`: today minus 1 year
   - `'custom'`: validates customRange is provided, normalizes both dates
   - Default: throws `Error('Unknown preset: ...')`

Add a date-only string guard: if input to `normalizeToLocalDate` is a string with `.length === 10` (e.g., `"2026-02-22"`), append `'T00:00:00'` before creating the Date to avoid the UTC-midnight trap where date-only ISO strings are parsed as UTC midnight per spec.

Include JSDoc comments for each function.

Do NOT add: any DOM manipulation, any imports from other files, any CSS. This is a pure logic module.

  <action>
Create `artha-cache.js` at the project root as an ES module.

**IndexedDB cache layer (INFRA-08):**

1. Import `openDB` from `'https://cdn.jsdelivr.net/npm/idb@8/+esm'` — this is the idb library by Jake Archibald (~1.19kB brotli'd).

2. Constants:
   - `DB_NAME = 'artha-cache'`
   - `DB_VERSION = 1`
   - `STORE_NAME = 'responses'`
   - `TTL_MS = 60 * 60 * 1000` (1 hour per user decision)

3. `getDB()` — Singleton pattern. Store the DB promise in a module-level `_db` variable. On first call, open the database with `openDB(DB_NAME, DB_VERSION, { upgrade(db) { db.createObjectStore(STORE_NAME, { keyPath: 'key' }) } })`. Return the cached promise on subsequent calls. This avoids opening a new connection per operation (a common anti-pattern that the research warns about).

4. `cacheGet(key)` — Async. Gets the record from the store by key. If not found, return `null`. If found, check `Date.now() - record.cachedAt > TTL_MS`. If stale, delete the record and return `null`. If fresh, return `record.data`. Use `db.get(STORE_NAME, key)` shorthand (each creates its own micro-transaction — safe across await boundaries per research).

5. `cachePut(key, data)` — Async. Stores `{ key, data, cachedAt: Date.now() }` using `db.put(STORE_NAME, ...)`. Overwrites any existing entry for that key.

6. `cacheInvalidate(key)` — Async. Deletes a single cache entry by key using `db.delete(STORE_NAME, key)`.

7. `cacheInvalidateAll()` — Async. Clears all entries using `db.clear(STORE_NAME)`. Used for manual refresh (the refresh button will call this).

8. Wrap the `getDB()` initialization in try/catch. If IndexedDB is unavailable (Safari private browsing edge case), set a module-level `_disabled = true` flag. When `_disabled` is true, `cacheGet` returns `null` and `cachePut`/`cacheInvalidate`/`cacheInvalidateAll` are no-ops. Log a single `console.warn('IndexedDB unavailable — caching disabled')`. This ensures the app works without caching rather than crashing.

Include JSDoc comments for each exported function.

Do NOT add: any DOM manipulation, any currency/date logic, any HTML. This is a pure cache module.
